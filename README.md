# Parks Canada Weather Station Optimization

This repository contains a reproducible analysis pipeline for evaluating whether any Parks Canada weather stations in Prince Edward Island National Park can be removed without materially reducing fire-weather monitoring coverage. The project integrates raw HOBOlink and ECCC weather data, cleans and standardizes the records, evaluates data fitness, benchmarks Park stations against the ECCC Stanhope reference station, computes daily FWI moisture codes, and produces evidence-based retention recommendations.

The final recommendation from the completed analysis is straightforward: retain all five Park stations. Each station contributes unique variance to the multivariate weather network, and removing any single station produces a material loss of information.

## Submission Files

Deliverables are collected in `deliverables/`:

- `deliverables/README.md` – quick guide to the final submission bundle
- `deliverables/cleaning.py` – consolidated ingestion and cleaning entry point
- `deliverables/analysis.ipynb` – documented EDA, PCA/clustering, FWI, and visual evidence notebook
- `deliverables/technical_report.md` – technical report draft for export to PDF or Word
- `deliverables/figures/` – selected figures used directly in the report and notebook

The original phased pipeline remains in `src/` and is the authoritative implementation.

## Repository Structure

- `data/raw/` – raw source files from Park stations and ECCC Stanhope
- `data/scrubbed/` – cleaned and modeled intermediate/final CSV artifacts
- `src/` – phased scripts for obtain, scrub, explore, and model stages
- `outputs/figures/` – figures generated by the original Phase 3 and Phase 4 scripts
- `docs/plans/` – phase summaries and implementation notes
- `deliverables/` – assignment submission bundle

## Environment Setup

Create and activate a Python environment, then install the required packages:

```powershell
python -m venv venv
.\venv\Scripts\Activate.ps1
pip install -r requirements.txt
```

## Reproducible Execution

Run the full workflow from the repository root:

```powershell
python deliverables/cleaning.py
python src/03_explore.py
python src/04_model.py
```

Then open `deliverables/analysis.ipynb` to review the documented analysis narrative and the final visuals.

## Implementation Notes

`deliverables/cleaning.py` is a thin wrapper that reuses the existing Phase 1 and Phase 2 scripts instead of duplicating logic. This keeps the assignment-friendly deliverable aligned with the tested project code.

Most report figures are reused directly from the completed project outputs. One exception is the original PCA biplot, which rendered as a near-empty plot in the generated artifacts; the submission bundle therefore uses a new station-profile clustering figure instead of relying on that broken visual.

## Recommendation Summary

- `NRW` shows the largest leave-one-out variance loss at 22.2%.
- `CAV`, `SBW`, `TRW`, and `GRE` each still exceed 16% variance loss when removed.
- All stations have bootstrap risk labels of `high` for removal.
- `GRE` remains important as a unique coastal site and as an FWI computation station.

These outputs support a do-not-consolidate recommendation for the current Park network.
